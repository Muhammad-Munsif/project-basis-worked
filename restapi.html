üìñ What is a RESTful API?

A RESTful API follows the constraints of REpresentational State Transfer architecture. It uses HTTP methods to access and manipulate resources.

üçΩÔ∏è Restaurant analogy

Menu = API endpoints ¬∑ Your order = request ¬∑ Food served = response.

Key REST principles

  • Client‚ÄëServer ‚Äì separation of concerns
  • Stateless ‚Äì each request carries all info
  • Cacheable ‚Äì responses declare cacheability
  • Uniform interface ‚Äì consistent resource naming
  • Layered system ‚Äì intermediate servers

Common HTTP status codes

Code Name Description
200 OK Request succeeded
201 Created Resource created
400 Bad Request Invalid input
404 Not Found Resource missing
500 Internal Error Server fault

üß© Endpoint structure & naming

Use nouns, plural forms, and nested routes for relations.

GET /api/users // all users
GET /api/users/42 // specific user
POST /api/users // create new
PUT /api/users/42 // update
DELETE /api/users/42 // delete
GET /api/users/42/posts // nested

üèóÔ∏è Build a simple API (Node.js)

const express = require('express');
const app = express();
app.use(express.json());
let users = [{ id:1, name:'John' }];

app.get('/api/users', (req,res) => res.json(users));
app.post('/api/users', (req,res) => {
  const user = { id: users.length+1, ...req.body };
  users.push(user); res.status(201).json(user);
});
app.listen(3000);

Test with Postman, curl, or the browser.
